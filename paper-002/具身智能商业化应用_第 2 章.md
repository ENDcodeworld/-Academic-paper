# 第 2 章 技术基础

## 2.1 VLA 模型核心技术

### 2.1.1 VLA 模型架构

视觉 - 语言 - 动作（Vision-Language-Action, VLA）模型是具身智能的核心技术，负责将视觉感知和语言理解转化为机器人动作。

**基本架构**：

```
┌─────────────────────────────────────────────────────────┐
│                    VLA 模型架构                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  输入层                                                  │
│  ├── 视觉输入：RGB 图像、深度图、点云                    │
│  ├── 语言输入：自然语言指令                             │
│  └── 本体感知：关节角度、速度、力矩                      │
│                                                         │
│  编码层                                                  │
│  ├── 视觉编码器：ViT、ResNet、CLIP                      │
│  ├── 语言编码器：LLaMA、T5、Transformer                 │
│  └── 感知融合：Cross-Attention、Concat                  │
│                                                         │
│  决策层                                                  │
│  ├── 任务理解：意图识别、目标解析                        │
│  ├── 动作规划：路径规划、技能选择                        │
│  └── 策略网络：动作概率分布                              │
│                                                         │
│  输出层                                                  │
│  └── 动作输出：关节角度、末端位姿、夹爪状态              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**主流架构对比**：

| 架构 | 代表模型 | 参数量 | 优势 | 劣势 |
|------|---------|--------|------|------|
| Transformer-based | RT-2、OpenVLA | 7B-70B | 性能强、泛化好 | 计算成本高 |
| CNN+RNN | RT-1、早期模型 | <1B | 效率高、易部署 | 性能有限 |
| 扩散模型 | π0、Diffusion Policy | 30B+ | 动作平滑、适合接触任务 | 推理慢 |
| 混合架构 | PaLM-E | 540B | 多模态融合好 | 成本极高 |

### 2.1.2 训练方法

**1. 行为克隆（Behavior Cloning）**

最常用的 VLA 训练方法，通过监督学习模仿专家演示。

**流程**：
```
专家演示数据集 D = {(o_1, a_1), (o_2, a_2), ..., (o_N, a_N)}
↓
最小化损失 L = Σ ||π(o_i) - a_i||²
↓
得到策略π*
```

**优点**：
- 简单直接
- 训练稳定
- 数据利用率高

**缺点**：
- 分布偏移问题
- 无法处理未见情况
- 需要大量标注数据

**2. 强化学习（Reinforcement Learning）**

通过与环境交互学习最优策略。

**常用算法**：
- PPO（Proximal Policy Optimization）
- SAC（Soft Actor-Critic）
- TD3（Twin Delayed DDPG）

**优点**：
- 可超越专家水平
- 适应动态环境
- 支持长期规划

**缺点**：
- 样本效率低
- 训练不稳定
- 安全顾虑

**3. 模仿学习 + 强化学习混合**

结合两者优势的主流方法。

**流程**：
```
阶段 1：行为克隆预训练 → 获得基础策略
↓
阶段 2：强化学习微调 → 优化性能
↓
阶段 3：人类反馈对齐 → 提升安全性
```

**代表工作**：
- RT-2：行为克隆预训练 + 多任务微调
- OpenVLA：大规模行为克隆
- π0：扩散模型 + 强化学习

### 2.1.3 关键技术突破

**1. 跨模态注意力机制**

实现视觉、语言、动作的深度融合。

```
Cross-Attention(Q, K, V) = softmax(QK^T/√d)V

其中:
Q = 语言 query
K, V = 视觉 key/value
```

**效果**：
- 细粒度特征对齐
- 动态权重分配
- 性能提升 15-20%

**2. 动作 tokenization**

将连续动作离散化，与语言统一建模。

**方法**：
```
连续动作 a ∈ R^7
↓ 离散化
a_token ∈ {0, 1, ..., 255}^7
↓ 与语言同一 vocab
统一 Transformer 处理
```

**优势**：
- 统一多模态输出
- 利用语言模型先验
- 支持组合动作

**3. 知识迁移**

从网络数据迁移知识到机器人控制。

**RT-2 案例**：
```
训练数据:
- 机器人轨迹：500K
- 网络图文：100M+

能力:
- 理解"可乐"是红色罐装饮料（从网络学习）
- 在桌上找到并抓取可乐（机器人技能）
- 零样本泛化到未见过的可乐品牌
```

### 2.3.2 仿真到真实迁移

**Sim2Real 技术**：

1. **域随机化**
   - 随机化纹理、光照、物理参数
   - 提升模型鲁棒性
   - 缩小 Sim2Real 差距 8-10%

2. **域自适应**
   - 对抗训练对齐域分布
   - 自训练利用真实数据
   - 缩小差距 10-12%

3. **真实数据微调**
   - 仿真预训练 + 真实微调
   - 最有效但需要真实数据
   - 缩小差距 12-15%

## 2.4 部署技术

### 2.4.1 模型优化

**量化**：
- FP32 → INT8：4× 压缩，<2% 性能损失
- FP16 → INT4：4× 压缩，<5% 性能损失

**剪枝**：
- 结构化剪枝：移除整个通道/层
- 非结构化剪枝：移除单个权重
- 压缩率 50-80%

**知识蒸馏**：
- 大模型 → 小模型
- 保持 95% 性能
- 压缩率 10-20×

### 2.4.2 实时系统

**延迟要求**：
- 高速操作：<50ms
- 一般操作：<100ms
- 慢速操作：<200ms

**优化技术**：
- 模型并行
- 流水线执行
- 预测性推理

### 2.4.3 安全监控

**运行时保障**：
- 异常检测
- 紧急停止
- 人机协作安全区
