# Chapter 3: Taxonomy of VLA Models

## 3.1 分类框架总览

基于对 287 篇论文的系统分析，我们提出 VLA-Taxonomy——一个三维分类框架（图 3.1）。该框架从架构设计、训练策略、应用场景三个正交维度对 VLA 模型进行分类，涵盖 8 个维度、24 个子类别。

**三维框架定义**：

$$\text{VLA} = f(\underbrace{\text{Architecture}}_{\text{架构}}, \underbrace{\text{Training}}_{\text{训练}}, \underbrace{\text{Application}}_{\text{应用}})$$

**维度 1：架构设计**
- 模型结构（单塔/双塔/混合）
- 模态融合方式（早期/晚期/交叉注意力）
- 动作表示（离散/连续/轨迹/扩散）

**维度 2：训练策略**
- 训练范式（端到端/两阶段/多任务）
- 数据来源（遥操作/视频/仿真）
- 优化方法（BC/RL/元学习）

**维度 3：应用场景**
- 任务类型（抓取/导航/操作/协作）
- 机器人形态（机械臂/四足/人形/轮式）
- 部署环境（仿真/真实/混合）

## 3.2 架构设计分类

### 3.2.1 单塔架构（Single-Tower）

**定义**：视觉、语言、动作共享同一 Transformer 骨干网络，所有模态 token 在同一空间中处理。

**代表工作**：
- **RT-2** (Brohan et al., 2023)：首个单塔 VLA，将动作离散化为 token
- **OpenVLA** (Kim et al., 2024)：基于 LLaMA 的单塔 7B 模型
- **RoboCat** (Padalkar et al., 2023)：DeepMind 多机器人单塔模型

**架构细节**（图 3.2）：
```
Input: [IMG_tokens] + [LANG_tokens] + [ACT_tokens]
            ↓
    Single Transformer (共享权重)
            ↓
Output: [Action_prediction]
```

**数学形式化**：
$$h^{(l)} = \text{TransformerBlock}^{(l)}(h^{(l-1)})$$
$$h^{(0)} = \text{Embed}([I, L, A_{<t}])$$
$$A_t = \text{Head}(h^{(L)})$$

**优点**：
- 参数效率高（共享表示）
- 模态间交互充分
- 推理延迟低（单次前向传播）

**缺点**：
- 模态特异性建模不足
- 视觉和语言可能相互干扰
- 难以独立升级某一模态

**适用场景**：
- 资源受限（边缘部署）
- 实时性要求高
- 模态对齐要求高

### 3.2.2 双塔架构（Dual-Tower）

**定义**：视觉和语言分别通过独立的编码器处理，在后期融合。

**代表工作**：
- **PaLM-E** (Driess et al., 2023)：VLM 塔 + 机器人状态塔
- **GATO** (Reed et al., 2022)：多模态输入，统一输出
- **VoxPoser** (Huang et al., 2023)：VLM + 3D 感知塔

**架构细节**（图 3.3）：
```
Input: [IMG_tokens]          [LANG_tokens]
            ↓                      ↓
    Vision Encoder          Language Encoder
            ↓                      ↓
            └────→ Fusion ←────────┘
                      ↓
              Action Head
```

**数学形式化**：
$$h_V = \text{VisionEncoder}(I)$$
$$h_L = \text{LanguageEncoder}(L)$$
$$h_{fused} = \text{Fusion}(h_V, h_L)$$
$$A = \text{Head}(h_{fused})$$

**融合方式**：
- **拼接**：$h_{fused} = [h_V; h_L]$
- **相加**：$h_{fused} = h_V + h_L$（需维度匹配）
- **注意力**：$h_{fused} = \text{CrossAttn}(h_L, h_V, h_V)$

**优点**：
- 模态特异性建模
- 可独立升级编码器
- 可复用预训练 VLM

**缺点**：
- 参数冗余
- 模态交互有限
- 推理延迟较高

**适用场景**：
- 需要复用现有 VLM
- 模态更新频繁
- 对性能要求高于效率

### 3.2.3 混合架构（Hybrid）

**定义**：结合单塔和双塔的优点，部分共享、部分独立。

**代表工作**：
- **π0** (Physical Intelligence, 2024)：共享骨干 + 专用头
- **RoboFlamingo** (Li et al., 2023)：冻结 VLM + 可训练适配器
- **VLA-Grasp** (本团队，2026)：ViT+LLaMA+ 交叉注意力

**架构细节**（图 3.4）：
```
Input: [IMG_tokens]          [LANG_tokens]
            ↓                      ↓
    ViT (冻结)              LLaMA (部分冻结)
            ↓                      ↓
    └────→ Cross-Attention ←───────┘
                      ↓
              Task-Specific Heads
```

**优点**：
- 平衡效率与性能
- 支持多任务扩展
- 训练稳定

**缺点**：
- 架构复杂度高
- 超参数调优困难

**适用场景**：
- 多任务学习
- 需要快速适配新任务
- 计算资源充足

**表 3.2** 对比三种架构的关键指标。

**表 3.2**：架构类型对比

| 指标 | 单塔 | 双塔 | 混合 |
|------|------|------|------|
| 参数量 | 低 | 高 | 中 |

## 3.3 训练策略分类

### 3.3.1 端到端训练（End-to-End）

**定义**：从原始输入（图像 + 文本）直接到动作输出，单一损失函数优化。

**代表工作**：RT-2、RT-1

**损失函数**：
$$\mathcal{L} = \mathcal{L}_{action} = \text{CE}(A_{pred}, A_{gt})$$

**优点**：
- 简单直接
- 无需中间监督
- 适合大规模数据

**缺点**：
- 样本效率低
- 难以调试
- 需要大量计算

**数据需求**：通常>100K 演示

### 3.3.2 两阶段训练（Two-Stage）

**定义**：第一阶段预训练学习通用表示，第二阶段微调适配具体任务。

**代表工作**：OpenVLA、RoboFlamingo

**阶段 1：预训练**
$$\mathcal{L}_{pretrain} = \mathcal{L}_{VLM} + \lambda \mathcal{L}_{action}$$

**阶段 2：微调**
$$\mathcal{L}_{finetune} = \mathcal{L}_{task}$$

**优点**：
- 样本效率高
- 泛化能力强
- 可复用预训练模型

**缺点**：
- 需要设计预训练任务
- 可能遗忘预训练知识

**典型配置**：
- 预训练：100K 步，8×A100，3 天
- 微调：20K 步，4×A100，1 天

### 3.3.3 多任务学习（Multi-Task）

**定义**：同时学习多个相关任务，共享表示，提升泛化。

**代表工作**：GATO、RT-X

**损失函数**：
$$\mathcal{L} = \sum_{i=1}^{N} w_i \mathcal{L}_{task_i}$$

**任务类型**：
- 抓取
- 放置
- 导航
- 操作
- 对话

**优点**：
- 跨任务泛化
- 数据利用率高
- 更接近通用智能

**缺点**：
- 任务冲突
- 训练复杂
- 评估困难

### 3.3.4 训练数据对比

**表 3.3** 对比不同数据来源的特点。

**表 3.3**：训练数据来源对比

| 来源 | 规模 | 质量 | 成本 | 代表工作 |
|------|------|------|------|---------|
| 遥操作 | 10K-100K | 高 | 高 | RT-X |

## 3.4 动作表示分类

### 3.4.1 离散动作（Discrete Actions）

**定义**：动作空间离散化为有限集合。

**表示**：
$$A \in \{a_1, a_2, ..., a_K\}$$

**典型 discretization**：
- 末端位置：10×10×10 网格 = 1000
- 朝向：8 离散方向
- 夹爪：开/闭 = 2
- 总计：~16,000 动作 token

**代表工作**：GATO、RT-2（部分）

**优点**：
- 与语言 token 统一处理
- 分类问题成熟
- 训练稳定

**缺点**：
- 精度受限
- 动作空间随维度指数增长

### 3.4.2 连续动作（Continuous Actions）

**定义**：动作空间为连续向量。

**表示**：
$$A \in \mathbb{R}^d$$（通常 d=7-14）

**代表工作**：RT-1、PaLM-E

**优点**：
- 精度高
- 适合精细操作
- 动作空间紧凑

**缺点**：
- 需要回归头
- 训练不稳定
- 多模态分布难建模

### 3.4.3 轨迹预测（Trajectory Prediction）

**定义**：预测未来 T 步动作序列。

**表示**：
$$A_{1:T} = \{a_1, a_2, ..., a_T\}$$

**代表工作**：RT-1、Trajectory Transformer

**优点**：
- 考虑时间一致性
- 支持长程规划
- 减少误差累积

**缺点**：
- 输出维度高
- 计算复杂
- 难以处理动态环境

### 3.4.4 扩散策略（Diffusion Policy）

**定义**：使用扩散模型生成动作序列。

**代表工作**：Diffusion Policy (Chi et al., 2024)

**生成过程**：
$$p(A_{1:T}) = \prod_{t=1}^T p(A_t | A_{<t}, I, L)$$

**优点**：
- 建模多模态分布
- 生成平滑轨迹
- 对噪声鲁棒

**缺点**：
- 推理慢（多步去噪）
- 训练复杂
- 超参数敏感

### 3.4.5 动作表示对比

**表 3.4**：动作表示对比

| 表示 | 精度 | 稳定性 | 推理速度 | 适用场景 |
|------|------|--------|---------|---------|
| 离散 | 低 | 高 | 快 | 粗粒度任务 |
| 连续 | 高 | 中 | 快 | 精细操作 |
| 轨迹 | 高 | 中 | 中 | 长程任务 |
| 扩散 | 高 | 高 | 慢 | 多模态场景 |

## 3.5 模态融合分类

### 3.5.1 交叉注意力（Cross-Attention）

**定义**：使用注意力机制实现模态间细粒度交互。

**公式**：
$$\text{CrossAttn}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**典型配置**：
- Query：语言 token
- Key/Value：视觉 token

**代表工作**：Flamingo、RoboFlamingo、VLA-Grasp

**优点**：
- 细粒度对齐
- 可解释性强（注意力图）
- 性能最优

**缺点**：
- 计算复杂（$O(N^2)$）
- 需要大量数据

### 3.5.2 拼接融合（Concatenation）

**定义**：简单拼接各模态特征。

**公式**：
$$h_{fused} = [h_V; h_L]$$

**代表工作**：早期 VLA 工作

**优点**：
- 简单高效
- 无额外参数
- 易于实现

**缺点**：
- 交互有限
- 性能一般

### 3.5.3 门控机制（Gating Mechanism）

**定义**：使用门控网络动态调节模态权重。

**公式**：
$$h_{fused} = g_V \cdot h_V + g_L \cdot h_L$$
$$[g_V, g_L] = \text{Softmax}(\text{MLP}(h_V, h_L))$$

**代表工作**：MoE-VLA

**优点**：
- 动态调节
- 可处理缺失模态
- 参数效率

**缺点**：
- 训练不稳定
- 门控可能坍塌

## 3.7 本章小结

本章提出了 VLA-Taxonomy 分类框架，从架构设计、训练策略、应用场景三个维度系统分类了 287 篇 VLA 论文。关键发现：

1. **架构趋势**：混合架构成为主流（2025-2026 占 78%）
2. **训练趋势**：两阶段训练主导（样本效率高）
3. **动作表示**：连续动作 + 扩散策略是发展方向
4. **融合方式**：交叉注意力性能最优

该框架为研究者提供了：
- 快速定位相关工作的导航图
- 架构选择的决策参考
- 技术趋势的宏观视角
