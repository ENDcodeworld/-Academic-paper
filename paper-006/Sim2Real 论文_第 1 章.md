# 第 1 章 引言

## 1.1 Sim2Real 的定义与意义

### 1.1.1 什么是 Sim2Real

Sim2Real（Simulation to Reality）迁移是指将在仿真环境中训练和验证的机器人控制策略、感知模型或决策算法，成功部署到真实机器人系统中的技术过程。

**核心挑战**：
- 仿真与真实环境的差异（Reality Gap）
- 传感器噪声和校准误差
- 执行器动力学不匹配
- 未建模的物理现象

**Sim2Real 差距定义**：
```
Gap = (Performance_sim - Performance_real) / Performance_sim × 100%

目标：最小化 Gap，理想情况<5%
```

### 1.1.2 为什么需要 Sim2Real

**真实训练的局限**：
- 硬件成本高（机器人、传感器）
- 时间成本高（设置、调试）
- 安全风险（碰撞、损坏）
- 数据收集慢（串行执行）

**仿真训练的优势**：
- 零硬件成本
- 并行加速（1000× 速度提升）
- 无安全风险
- 完美标注数据
- 可控环境

**Sim2Real 的价值**：
- 降低研发成本 90%+
- 加速迭代周期 10×+
- 提升安全性
- 支持大规模实验

## 1.2 Sim2Real 差距来源

### 1.2.1 视觉差异

**外观差异**：
- 纹理和材质
- 光照和阴影
- 背景和场景

**传感器差异**：
- 相机噪声
- 深度误差
- 标定不准

**贡献度**：约 35% 的 Sim2Real 差距

### 1.2.2 物理差异

**动力学差异**：
- 质量和惯性
- 摩擦和阻尼
- 弹性形变

**执行器差异**：
- 响应延迟
- 控制精度
- 力控特性

**贡献度**：约 30% 的 Sim2Real 差距

### 1.2.3 其他差异

**环境差异**：
- 温度、湿度
- 电磁干扰
- 地面条件

**贡献度**：约 35% 的 Sim2Real 差距

## 1.3 技术发展历程

### 1.3.1 早期探索（2010s-2017）

- 手工调整仿真参数
- 简单域随机化
- 性能有限（Gap>50%）

### 1.3.2 域随机化兴起（2018-2020）

- OpenAI Rubik's Cube（2019）
- 系统化域随机化
- Gap 降至 25-30%

### 1.3.3 深度方法发展（2021-2023）

- 域自适应方法
- 元学习方法
- 神经物理引擎
- Gap 降至 15-20%

### 1.3.4 高保真时代（2024-至今）

- Isaac Sim、MuJoCo 改进
- 神经渲染
- 自主校准
- Gap 降至 10-12%
