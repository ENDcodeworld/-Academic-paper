## 第 2 章 安全基础理论（10,000 字）

### 2.1 AI 风险分类与评估

#### 2.1.1 风险分类框架

**按风险来源分类**：

**1. 设计风险（Design Risks）**
- 目标函数设计错误
- 约束条件缺失
- 训练数据偏见
- 架构缺陷

**2. 部署风险（Deployment Risks）**
- 环境不匹配
- 用户误用
- 对抗攻击
- 系统故障

**3. 使用风险（Usage Risks）**
- 恶意使用
- 过度依赖
- 责任逃避
- 技能退化

**4. 系统性风险（Systemic Risks）**
- 市场垄断
- 就业冲击
- 信息生态破坏
- 地缘政治影响

**按风险严重程度分类**：

| 等级 | 描述 | 例子 | 应对 |
|------|------|------|------|
| L1 轻微 | 个体不便 | 错误建议 | 用户教育 |
| L2 中等 | 个体损失 | 财务损失 | 赔偿机制 |
| L3 严重 | 群体伤害 | 歧视决策 | 监管干预 |
| L4 灾难 | 社会动荡 | 大规模失业 | 系统性应对 |
| L5 存在性 | 人类威胁 | 超级智能失控 | 全球协作 |

### 2.2 对齐问题形式化

#### 2.2.1 数学框架

**基本设定**：

```
人类意图：H（未知）
AI 目标函数：R（可设计）
AI 策略：π
世界状态：s
动作：a

目标：找到π* 使得 E[H(s,a)] 最大化
约束：π 必须安全、可解释、可控
```

**外层对齐形式化**：

```
给定人类偏好数据 D = {(σ₁, σ₂, y)}
其中 σ 是轨迹，y 是偏好标签

学习奖励函数 R 使得:
R(σ₁) > R(σ₂) 当且仅当 人类偏好σ₁

优化：min L(R, D) + λ·Safety(R)
```

**内层对齐形式化**：

```
给定训练目标 R_outer
模型学习内部目标 R_inner

问题：R_inner ≈ R_outer ?

检测：机制分析、探针
缓解：正则化、约束
```

#### 2.2.2 价值学习

**偏好学习**：
- 从人类偏好学习奖励函数
- 方法：比较学习、逆强化学习
- 挑战：偏好不一致、动态变化

**逆强化学习（IRL）**：
- 从专家演示学习奖励
- 方法：最大熵 IRL、GAIL
- 挑战：多解性、计算复杂

**合作逆强化学习（CIRL）**：
- 人机协作学习
- 人类主动教学
- 更高效但需要人类参与

### 2.3 安全约束理论

#### 2.3.1 约束类型

**硬约束（Hard Constraints）**：
- 必须满足
- 违反不可接受
- 例子：不伤害人类

**软约束（Soft Constraints）**：
- 尽量满足
- 允许权衡
- 例子：效率 vs 安全

**条件约束（Conditional Constraints）**：
- 特定条件下激活
- 上下文相关
- 例子：紧急情况下可突破常规

#### 2.3.2 约束实现方法

**拉格朗日方法**：
```
L(π, λ) = E[R(π)] - λ·E[C(π)]
其中 C 是约束违反程度
```

**屏障函数方法**：
```
在约束边界设置无限惩罚
保证不越界
```

**安全层方法**：
```
在策略输出后添加安全过滤
确保动作安全
```

**验证方法**：
- 模型检测
- 定理证明
- 抽象解释
- 运行时验证

### 2.4 可解释性基础

#### 2.4.1 可解释性类型

**全局可解释性**：
- 理解模型整体行为
- 方法：特征重要性、规则提取
- 用途：模型审计、调试

**局部可解释性**：
- 理解单个预测
- 方法：LIME、SHAP
- 用途：个案解释、申诉

**事后可解释性**：
- 训练后解释
- 灵活但可能不准确
- 主流方法

**内在可解释性**：
- 模型本身可解释
- 如决策树、线性模型
- 性能可能受限

#### 2.4.2 解释方法

**注意力可视化**：
- 显示模型关注区域
- 直观但可能误导
- Transformer 常用

**特征归因**：
- LIME：局部线性近似
- SHAP：基于博弈论
- 归因一致性重要

**反事实解释**：
- "如果 X 改变，结果会如何"
- 直观易懂
- 计算成本高

**示例基解释**：
- 展示相似案例
- 案例基推理
- 法律领域适用

### 2.5 鲁棒性与对抗防御

#### 2.5.1 对抗攻击类型

** evasion 攻击**：
- 测试时扰动输入
- 误导模型预测
- 例子：对抗样本

**投毒攻击**：
- 训练时污染数据
- 植入后门
- 例子：数据投毒

**模型窃取**：
- 查询模型窃取参数
- 知识产权损失
- 例子：模型提取

**成员推断**：
- 推断训练数据成员
- 隐私泄露
- 例子：成员攻击

#### 2.5.2 防御方法

**对抗训练**：
- 训练时加入对抗样本
- 提升鲁棒性
- 最常用方法

**输入预处理**：
- 去噪、压缩
- 破坏对抗扰动
- 简单有效

**随机化防御**：
- 随机 dropout
- 随机输入变换
- 增加攻击难度

**形式化验证**：
- 数学证明鲁棒性
- 强保证
- 计算成本高

## 第 3 章 对齐技术方法（12,000 字）

### 3.1 监督微调（SFT）

#### 3.1.1 方法原理

**基本流程**：
```
1. 收集高质量标注数据
2. 在预训练模型上微调
3. 学习遵循指令
4. 评估对齐程度
```

**数据要求**：
- 指令 - 回复对：10,000-100,000
- 高质量人工标注
- 多样性覆盖
- 安全相关内容

**训练技巧**：
- 学习率调度和 warmup
- 梯度累积
- 混合精度训练
- 早停防止过拟合

#### 3.1.2 性能与局限

**优势**：
- 简单直接
- 训练稳定
- 效果可预测

**局限**：
- 数据成本高
- 扩展性差
- 无法学习隐含偏好
- 可能过拟合标注风格

**典型性能**：
- 指令遵循：80-85%
- 安全性：75-80%
- 有用性：80-85%

### 3.2 人类反馈强化学习（RLHF）

#### 3.2.1 方法原理

**三阶段流程**：

**阶段 1：SFT 预训练**
```
在指令数据上微调
获得基础策略π_SFT
```

**阶段 2：奖励模型训练**
```
收集人类偏好数据：(prompt, response_A, response_B, preference)
训练奖励模型 R 预测人类偏好
优化：max E[R(prompt, response)]
```

**阶段 3：RL 优化**
```
使用 PPO 等算法优化策略
目标：max E[R(prompt, π(prompt))] - β·KL(π||π_SFT)
KL 散度防止偏离太多
```

#### 3.2.2 关键技术

**偏好数据收集**：
- 标注员培训
- 质量把控
- 多样性保证
- 规模：50,000-500,000 比较

**奖励模型设计**：
- 架构：通常用 Transformer
- 训练：成对比较损失
- 评估：准确率、一致性
- 校准：避免过度自信

**RL 算法选择**：
- PPO：最常用，稳定
- A2C：简单但方差大
- SAC：连续动作空间
- 定制算法：针对 LLM 优化

#### 3.2.3 性能与挑战

**性能**：
- 指令遵循：85-90%
- 安全性：85-90%
- 有用性：85-90%
- 人类偏好：显著优于 SFT

**挑战**：
- 训练复杂，需要调参
- 不稳定，可能崩溃
- 计算成本高
- 奖励黑客风险

**改进方向**：
- 更好的奖励模型
- 更稳定的 RL 算法
- 减少人工标注
- 多目标优化

### 3.3 直接偏好优化（DPO）

#### 3.3.1 方法原理

**核心洞察**：
- RLHF 复杂，需要奖励模型和 RL
- 可以直接从偏好数据优化策略
- 数学等价但更简单

**DPO 损失函数**：
```
L_DPO = -E[log σ(β·log(π(y_w|x)/π_ref(y_w|x)) - β·log(π(y_l|x)/π_ref(y_l|x)))]

其中:
- y_w: 人类偏好的回复
- y_l: 人类不偏好的回复
- π_ref: 参考模型（通常 SFT）
- β: 温度参数
```

**优化**：
- 直接梯度下降
- 无需 RL
- 无需奖励模型
- 稳定高效

#### 3.3.2 性能对比

**vs RLHF**：

| 维度 | RLHF | DPO |
|------|------|-----|
| 性能 | 90% | 88-90% |
| 稳定性 | 中 | 高 |
| 复杂度 | 高 | 低 |
| 计算成本 | 高 | 中 |
| 调参难度 | 难 | 易 |

**优势**：
- 简化训练流程
- 更稳定
- 计算效率高
- 易于复现

**局限**：
- 理论假设强
- 对数据质量敏感
- 可能不如 RLHF 灵活

### 3.4 宪法 AI 与自对齐

#### 3.4.1 方法原理

**核心思想**：
- 用 AI 自己进行对齐
- 减少人工标注依赖
- 可扩展到大规模

**两阶段流程**：

**阶段 1：监督学习**
```
1. 定义"宪法"（原则列表）
2. AI 生成回复
3. AI 根据宪法自我批评
4. AI 修改回复
5. 在修改数据上微调
```

**阶段 2：强化学习**
```
1. AI 生成多个回复
2. AI 根据宪法评估
3. 选择最佳回复
4. RL 优化偏好
```

#### 3.4.2 宪法设计

**宪法内容**：
- 无害性原则
- 诚实性原则
- 帮助性原则
- 公平性原则
- 隐私保护原则

**设计考虑**：
- 清晰明确
- 可操作
- 覆盖全面
- 无冲突

**示例**：
```
1. 回复不应鼓励暴力或仇恨
2. 回复不应提供危险信息
3. 回复应承认不确定性
4. 回复应尊重隐私
5. 回复应避免歧视
```

#### 3.4.3 性能与前景

**性能**：
- 接近 RLHF 水平
- 安全性相当
- 有用性略低
- 可扩展性好

**优势**：
- 减少人工依赖
- 可扩展
- 透明可审计
- 原则驱动

**挑战**：
- 宪法设计困难
- 自我评估可能不可靠
- 需要基础对齐
- 长期效果待验证

当前，对齐技术从 RLHF 发展到 DPO、宪法 AI，性能显著提升；安全评估从单一指标发展到综合基准；治理框架从讨论走向实施。但可扩展监督、可解释对齐、多智能体安全等核心挑战仍未解决。

中国 AI 安全发展面临机遇与挑战并存。市场规模大、应用场景丰富、政策支持力度大是优势；核心技术、高端人才、标准制定是短板。需要政府、企业、学术界协同努力，走出一条发展与安全并重的道路。

AI 安全的终极目标是确保人工智能始终服务于人类利益。这需要技术创新、制度完善、社会共识的共同努力。我们正处于 AI 发展的关键时期，今天的决策将影响人类的未来。让我们携手推进 AI 安全研究，创造安全、可靠、有益的人工智能未来。
